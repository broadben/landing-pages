MCP wasn't built for production. Conduit is.

🔓

No auth

Most MCP servers ship wide-open. The ones that don't roll their own bearer-token check. Either way, there's no shared concept of identity, scopes, or rotation.

🌑

No visibility

When an agent does the wrong thing across fourteen tool calls, debugging is grep against console output. There's no trace, no replay, no audit log.

🚨

No control

A buggy agent can hammer an upstream API for hours before anyone notices. There's no rate limit, no kill switch, no cost cap.

One proxy. Every agent. Every server.

Point your agent at Conduit instead of your MCP servers directly. Conduit handles authentication, enforces rate limits, captures every JSON-RPC call, and surfaces it in a dashboard you already know how to read.

FAQ

Will Conduit work with my MCP server?

Yes, if your server speaks MCP over stdio or streamable HTTP. Both transports are supported.

Do I have to change my agent code?

No. Conduit speaks the same wire protocol as any MCP server. Point your agent at Conduit's address and it works.

Will this slow my agent down?

Conduit adds 1–3ms of latency per call on average. Tools that already take 100–2000ms see negligible impact.

Why Apache 2.0?

Permissive licensing is the distribution strategy. We have no plans to relicense. The OSS will always have feature parity for individual and small-team use.

What's the difference between Conduit and Cloudflare AI Gateway?

Cloudflare's gateway is built for LLM API calls (OpenAI, Anthropic, etc.) and is closed source. Conduit is MCP-native, open source, and self-hostable.

Can I run Conduit behind my corporate VPN?

Yes. Self-host gives you full network control. Conduit Cloud Enterprise also supports private deployments.

Do you store my tool-call arguments and responses?

Self-host: stored locally, you control it. Cloud: stored encrypted in your tenant, retention per your plan. You can configure redaction rules for sensitive fields.

Is there an enterprise version?

Yes. Contact us for SOC 2, BAA, on-prem deployment, and custom integrations.
